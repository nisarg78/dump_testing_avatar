1. Executive Summary

What is MechAvatar? A Retrieval-Augmented Generation (RAG) platform that transforms 25,000+ mainframe documents — COBOL source, JCL procedures, BDD table definitions, functional specs, dump files — into an intelligent knowledge base that developers, QA testers, and support engineers can query in natural language.

Why RAG instead of fine-tuning? MECH documentation changes with every release (new code patches, operational updates, table modifications). RAG retrieves from a live index that updates in minutes. Fine-tuning requires days of retraining with no guarantee the model learns identifiers like BDDDABI or TWASPDA. RAG also provides exact source attribution — every answer cites the document, section, and page it came from.

5. Ingestion Pipeline — Step Function State Machine

Each document uploaded to S3 triggers one Step Function execution. The state machine orchestrates extraction, embedding, and indexing with built-in retry, error handling, and a dedicated path for skipped documents (empty DOCX, image-only files).

STEP FUNCTION FLOW — Per Document
Critical s3_key Forwarding: The s3_key field is forwarded from L02 extraction_result → through the Step Function state → L03 → L04. Lambda 04 uses it to compute source_priority (1-9) and s3_folder. Without this forwarding, every document defaults to priority=100 and the search boost system (2×/1.5×/1.2×) is completely inactive.
Skipped Document Path (NEW): If L02 returns status: "skipped" (empty body, image-only DOCX), the CheckIfSkipped state routes to HandleSkippedDocument → DocumentSkippedComplete (Succeed). Previously this crashed the SF because the embedding step received null chunk data.
Batch Handoff (L03): Documents with >975 chunks: L03 processes one batch and returns statusCode 202 (not RuntimeError). The SF embedding_batch_keys array ensures L04 loads ALL batch files. Previously L04 only indexed the last batch — causing silent data loss.

9.4 Why Hybrid > Pure Semantic

Semantic alone misses exact identifiers. "Show me BDDDABI table structure" — the embedding for this query is close to "database table" generically, but BM25 on the bdd_tables field hits the exact chunk containing BDDDABI. Without BM25, the top results might be any table documentation, not the specific BDD macro.

10. Reranking, Grounding & Confidence

10.1 Multi-Stage Reranking Pipeline

StageMethodWhat It DoesCost
6: DiversityIn-memoryCollects best chunk per unique document first. Prevents one high-scoring file from dominating all slots. Critical for cross-document synthesis.~1ms · 0 API
7a: EntityIn-memoryBoosts chunks containing query entities. Score × (1 + 0.1 × entity_count × 1.5). Fast deterministic boost applied before expensive cross-encoder.~5ms · 0 API
7b: Cross-EncoderHaiku LLMHaiku scores each of top 12 chunks 0.0-1.0 for direct relevance. Final = 0.4 × retrieval + 0.6 × cross_encoder. Chunks beyond 12 get neutral 0.5.~800ms · 1 API
8: CompressHaiku LLMRemoves sentences from each chunk that aren't relevant to the query. Max 4,000 compressed tokens. Reduces noise in Sonnet context window.~300ms · 1 API
9: ReorderIn-memoryLost-in-middle mitigation: best chunks at start and end of context. LLMs attend more to beginning and end of long inputs.~1ms · 0 API

10.2 Grounding Verification

StatusMeaningAction
fully_groundedAll claims found in contextAnswer returned as-is
partially_groundedMost claims found, some gapsNote about missing info appended
weakly_groundedClaims not clearly in contextWarning: "may contain unverified info"
ungroundedFabricated / hallucinatedStrong warning. Confidence set <0.5

10.3 Confidence Formula

Final confidence = search_conf × 0.6 + groundedness × 0.4
Where search_conf = avg_retrieval_score × 0.6 + source_coverage × 0.2 + intent_confidence × 0.2
Normalized from raw OpenSearch scores (2-30 range) to 0.0-1.0 scale.

15.6 Re-Index Procedure

  1. Invoke L04: {"action":"cleanup","clear_pipeline":true}
  2. L04 auto-recreates indexes with correct mapping
  3. Re-upload all documents to S3 (or L05 reprocess_all)
  4. Monitor DynamoDB until all status=indexed
  5. Test: 5 sample questions → verify citations, source_priority, confidence
